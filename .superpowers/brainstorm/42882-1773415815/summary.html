Finalizing DocOCR Design

Here is the consolidated design based on our conversation. Let me know if this looks correct before I write up the formal spec.

1. User Interface: Split View

A Kotlin Multiplatform (Compose M3) web application featuring a dual-panel layout:

  • Left Panel: Drag-and-drop zone for uploading documents (PDFs, Images). Displays a preview of the uploaded content.
  • Right Panel: Displays the extracted OCR text results for easy side-by-side comparison with the original image.
  • State: Clear loading indicators while the document is being processed.

2. Architecture & Backend

The KMP frontend will communicate with an existing backend infrastructure:

  • Frontend: KMP Web App (`wasmJs` / `js` targets)
  • Backend: Existing Java Spring Boot application hosted on AWS ECS, fronted by an Application Load Balancer (ALB).
  • OCR Engine: The Spring Boot backend will be responsible for securely interacting with the AWS Textract API.
  • API Contract: The KMP app will send the file (multipart/form-data or base64) to a Spring Boot endpoint and receive JSON containing the extracted text.
